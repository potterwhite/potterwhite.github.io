项目概述

考勤分析器是一个为企业内部开发的效率工具，旨在解决复杂且耗时的员工考勤数据分析任务。它能够将原始的Excel导出数据，通过一个高度灵活、由规则驱动的引擎，转化为清晰、可执行的分析报告，极大地提升了HR部门的工作效率。

核心功能与技术栈

  • 规则驱动引擎: 所有的分析逻辑都由一个简单的 rules.toml 文件控制，使得适应政策变化无需修改代码。
  • 全面的分析指标: 精确计算出勤天数、各类加班时长、迟到、早退，并独立追踪缺卡与缺勤。
  • 双视图报告系统: 可同时生成用于通用布告的“公开版报告”和用于深入审计的“详细版报告”。
  • 技术栈: Python (后端逻辑), Pandas (数据处理), Streamlit (Web界面), TOML (配置管理)。

核心算法设计 (V1.6)

作为本项目的设计者，我定义了一套严谨的考勤分析算法，它是整个工具的“大脑”。这套算法的核心思想是客观、公平地量化考勤事件，其主要规则如下：

1. 工作单元状态判定

每个工作单元（半天）会被精确判定为以下三种状态之一：

  • 有效出勤: 上下班均有有效打卡。
  • 缺勤: 上下班均缺卡。
  • 异常: 其他任何情况（如单次缺卡）。

2. “有效出勤”优先原则

所有关于迟到、早退和自愿加班的计算，都只在“有效出勤”的单元内进行，确保了数据的严谨性。

3. 严重迟到处理

当迟到超过设定的阈值时，该工作单元从“实际打卡时间”到“应下班时间”的整个时间段，将被计算为加班时间。

4. 灵活的加班计算

总加班时长由两部分构成：“自愿加班”（下午下班后，超过延迟门槛的时间）和“因严重迟到产生的加班”。

项目背后的故事

本项目是现代AI辅助开发模式的一次成功实践。约95%的代码是在与大语言模型（LLM）的协作下生成的。在此过程中，我的角色从传统的“编码者”转变为项目的**“架构师、导演和品控负责人”**。

这种人机协作的伙伴关系，将原本预估需要1-2周的开发周期，显著缩短到了几天之内，展示了一种快速、高质量软件创造的新范式。
