Project Overview

The Attendance Analyzer is an in-house efficiency tool developed to tackle the complex and time-consuming task of analyzing employee attendance data within a corporate environment. The application transforms raw Excel exports into clear, actionable reports via a highly flexible, rule-driven engine, significantly improving HR workflow efficiency.

Core Features & Tech Stack

  • Rule-Driven Engine: All analysis logic is controlled by a simple rules.toml file, allowing for easy adaptation to policy changes without modifying the source code.
  • Comprehensive Metrics: Accurately calculates attended days, various types of overtime, lateness, early departures, and independently tracks missing punches and absent days.
  • Dual-View Reporting: Generates both a high-level "Public Report" for general announcements and a "Detailed Report" for in-depth audits.
  • Tech Stack: Python (backend logic), Pandas (data manipulation), Streamlit (web interface), TOML (configuration).

Core Algorithm Design (V1.6)

As the project's architect, I designed a rigorous attendance analysis algorithm that serves as the "brain" of the tool. The core principle of this algorithm is to quantify attendance events objectively and fairly, based on the following key rules:

1. Work Unit State Determination

Each work unit (half-day) is precisely classified into one of three states:

  • Attended Unit: Both on-duty and off-duty punches are valid.
  • Absent Unit: Both on-duty and off-duty punches are missing.
  • Abnormal Unit: Any other combination (e.g., a single missing punch).

2. "Attended Unit First" Principle

All advanced calculations for lateness, early quits, and voluntary overtime are performed only within "Attended Units", ensuring data integrity.

3. Severe Lateness Handling

When lateness exceeds a defined threshold, the entire duration of that work unit, from the actual punch-in time to the scheduled punch-out time, is calculated as overtime.

4. Flexible Overtime Calculation

Total overtime is composed of two parts: "Voluntary Overtime" (time worked after the afternoon shift, beyond a delay threshold) and "Overtime from Severe Lateness".

The Story Behind The Project

This project is a successful case study in modern, AI-assisted development. Approximately 95% of the code was generated in collaboration with a Large Language Model (LLM). Throughout this process, my role evolved from a traditional "coder" to that of the project's Architect, Director, and Quality Assurance Lead.

This human-AI partnership significantly reduced the development lifecycle from an estimated 1-2 weeks to just a few days, demonstrating a new paradigm for rapid, high-quality software creation.

This project was developed as an internal corporate asset, and its source code is not publicly available.
